# FriendsGacha 仕様書（管理者用メモ）

このファイルは実装済みの仕様を自分用にまとめたものです。コードの詳細はソースを正としてください（各見出しに対応ファイルを記載）。

## 遊び部分

### ガチャの仕様

対応ファイル: [`js/gacha.js`](../js/gacha.js), [`js/rarity.js`](../js/rarity.js)

- **通貨**: マメ（[マメ工房](#マメ工房の仕様)で貯める、上限999,999,999）
- **消費コスト**: 1回引く = 5マメ / 10連引く = 50マメ
  - 10連は現在UI導線を非表示にしている（ロジック自体は残してあり、`#drawTenBtn`の`hidden`クラスを外せば復活可能）
- **抽選ロジック**:
  - レアリティを重み付き抽選（`RARITY`の`weight`: N=80% / R=18% / SR=1.8% / SSR=0.2%、カタログに存在するレアリティのみで正規化）
  - 選ばれたレアリティ内のアイテムから完全ランダムで1点抽選
- **フロー**:
  1. 「引く」ボタン → カタログ読込 → マメ消費（不足していれば「マメが足りません」アラートで中断）→ コレクションに記録 → パックが開封待ち状態に
  2. パックをタップ → 開封演出 → カード表示 → 「もう一度引く」ボタン表示
  3. 「もう一度引く」を押すと、演出を挟まず**即座に次の1回を引き直す**（直前が通常ガチャなら通常ガチャを、ナゾマメガチャならナゾマメガチャを繰り返す。マメ／ナゾマメが足りなければアラートを出してパック選択画面に戻り、対応するボタンは残高不足で無効化される）
- 抽選結果はブラウザのIndexedDB（`collection`ストア）に所持記録として保存される（`js/db.js`）

### ナゾマメガチャの仕様

対応ファイル: [`js/gacha.js`](../js/gacha.js), [`js/rarity.js`](../js/rarity.js)

- 「ナゾマメで引く」ボタンから引く特別なガチャ。消費はマメではなく**ナゾマメ10個**
- 抽選対象は**R・SR・SSRのみ**（Nは出ない、R以上確定）。専用オッズ R70% / SR25% / SSR5%（`NAZO_WEIGHTS`、カタログに存在するレアリティのみで正規化）
- カタログにR以上のアイテムが1つも登録されていない場合は「R以上のイラストがまだ登録されていません。」とアラートして中断（ナゾマメは消費されない）
- 演出・カード表示・コレクション記録は通常ガチャと共通のロジックを流用
- **現在一時的に無効化中**: `js/gacha.js`の`NAZO_GACHA_ENABLED`を`false`にしており、ボタンは表示されるが常に無効化（押せない）。再開する場合は`true`に戻すだけでよい

### マメ工房の仕様

対応ファイル: [`js/mame.js`](../js/mame.js)

- ブラウザごとの放置生産要素。IndexedDBの`settings`ストア（キー`'mame'`）に状態を保存
- **マメの所持上限**: 999,999,999（生産・オフライン進行いずれもこの値でクランプされる）
- **生産ライン**: 初期1本。5秒ごとに+1マメを自動生産（バッファ上限なし。貯まる量そのものはマメの所持上限まで）
- **タップ**: 生産ラインをタップすると、そのラインの残り時間が即座に短くなる
  - 初期状態は残り時間が「1/2」に短縮。「タップの効果」を強化するたびに 1/3 → 1/4 …と短縮率が強くなる
- **オフライン進行**: ページを開いた瞬間に前回訪問からの経過時間をまとめて計算し、その分のマメを即加算
- クリティカル演出は削除済み（生産量は常に一定）
- **アップグレード**（マメを消費して強化、コストは購入するたびに指数的に上昇。Lv上限あり、上限に達すると「MAX」表示でそれ以上強化できない）:
  | アップグレード | 効果 | 初期コスト / 上昇率 | Lv上限 |
  |---|---|---|---|
  | 製造速度 | 生産間隔を短縮（間隔 ×0.85 per Lv、最短0.1秒でクランプ） | 15マメ / ×1.35 | Lv50 |
  | 一度に生産する量 | 1回の完成量アップ（1→2→…→1,000,000の20段階テーブル） | 25マメ / ×2 | Lv19（テーブル最終値=100万個で頭打ち） |
  | 並列生産ライン | ラインを1本追加（同時生産数が増える） | 150マメ / ×2 | Lv19（ライン数20本で頭打ち） |
  | タップの効果 | タップ時の残り時間短縮率を強化（1/2→…→1/11） | 20マメ / ×1.8 | Lv9（短縮率1/11で頭打ち） |
  - ※`docs/Balance.md`のLv表記（Lv50/Lv20/20本/Lv10）は内部的には0始まりのレベルカウントのため実装上の上限レベル値と1ずれている項目があるが、最終到達値（0.1秒・100万個・20本・1/11）はすべて仕様どおり
- **ナゾマメ交換**: マメ工房の交換セクションから、1億マメ→ナゾマメ1個に手動で交換できる（自動変換はしない）。ナゾマメの所持上限は9,999
- **今回のスコープ外**（意図的に未実装、将来の拡張候補）:
  - オーバーチャージ（100%超えまで待って回収すると倍率アップ）: 「完成したら即座に次の生産が始まる」という現行仕様と相性が悪いため見送り

### コレクションの仕様

対応ファイル: [`js/collection.js`](../js/collection.js)

- 達成率を「所持数 / 総アイテム数」で表示
- レアリティフィルタ（ALL / N / R / SR / SSR）
- 未所持アイテムはシルエット＋「？」表示、所持済みはサムネイル＋所持数バッジ表示
- アイテムをタップすると詳細モーダル（名前・作者・フレーバーテキスト・所持数・初取得日）を表示。未所持なら「？？？？？」と入手を促す文言のみ表示

### その他の共通UI

対応ファイル: [`index.html`](../index.html), [`js/main.js`](../js/main.js)

- ヘッダーの「？」ボタン → 遊び方ポップアップ（ガチャを引く／マメの製造／ナゾマメガチャ／コレクションの4項目を説明）
  - **初回訪問時は自動的に開く**（IndexedDBの`settings`ストア、キー`'howToPlaySeen'`で既読管理）
- ヘッダーに🫘マメ・🔮ナゾマメの残高バッジを常時表示
- 新イラスト追加通知ポップアップ「🎉 新しいイラストが追加されました！」
  - `data/items.json`のアイテムID一覧をIndexedDB（キー`'knownItemIds'`）に記録し、前回訪問時より増えていたら表示
  - **遊び方ポップアップをまだ見ていない人（＝初回訪問者）には表示しない**（チュートリアルより新着通知が先に出るのを防ぐため）
- フッター: 左下にバージョン表記（`v1.0.0`固定テキスト、更新時は`index.html`を手動で書き換える）、右下にコピーライト
- 更新のお知らせポップアップ（`data/releasenote.json`の`title`/`txt`を表示）
  - IndexedDB（キー`'checkRereleaseNote'`）に「最後に見た`version`」を記録し、`data/releasenote.json`の`version`と一致しなければ表示。表示後は現在の`version`を記録して既読にする
  - **遊び方ポップアップをまだ見ていない人（＝初回訪問者）には表示しない**（バージョンの記録自体は行うが、ポップアップは出さない）
  - 管理者がお知らせを見せ直したい場合は`version`を上げて公開する（詳細は[運営部分](#更新のお知らせの出し方リセット方法)を参照）

## 運営部分

### 更新のお知らせの出し方・リセット方法

対応ファイル: [`data/releasenote.json`](../data/releasenote.json), [`js/main.js`](../js/main.js)

`data/releasenote.json`は次の形式:

```json
{
  "version": 1,
  "title": "更新のお知らせ",
  "txt": "本文。\n で改行できる"
}
```

新しい更新をプレイヤーに知らせたいときの手順:

1. `title`と`txt`を新しい内容に書き換える（`txt`内の改行は`\n`で表現する）
2. `version`の数値を1つ上げる（例: 1 → 2）
3. `git add` → `git commit` → `git push`

プレイヤー側は「最後に見た`version`」をIndexedDBに保持しているだけなので、**`version`を上げて公開するだけで、既読状態が自動的にリセットされ全員に再表示される**（個別のリセット操作は不要）。`title`/`txt`だけ変えて`version`を上げ忘れると、既読済みのプレイヤーには表示されないので注意。

### イラストの追加方法

対応ファイル: [`admin.html`](../admin.html), [`js/admin.js`](../js/admin.js)

1. `admin.html` をブラウザで開く（初回はブラウザのIndexedDBが空の場合、公開中の`data/items.json`を自動読み込みする）
2. フォームに入力して「追加する」を押す
   - 名前（必須）／レアリティ（必須、N・R・SR・SSR）／作者名／フレーバーテキスト／イラスト画像（必須、正方形に近い画像推奨）
   - 画像はBase64データURLとしてそのままアイテムデータに埋め込まれる（＝画像ファイルを別途サーバーに置く必要はないが、`items.json`が大きくなりがちなので画像サイズに注意）
   - **この時点ではまだこのブラウザ内（IndexedDB）にしか保存されておらず、公開サイトには反映されない**
3. 一覧から既存アイテムの「編集」「削除」も可能（同じくローカル反映のみ）
4. **公開サイトに反映する手順**:
   1. 「カタログを書き出す (items.json)」ボタンで`items.json`をダウンロード
   2. リポジトリの `data/items.json` をダウンロードしたファイルで上書き
   3. `git add` → `git commit` → `git push`（GitHub Pagesへの反映を確認）
5. 公開後、既にサイトを訪れたことがある（遊び方ポップアップ既読の）ユーザーには、次回アクセス時に自動で「新しいイラストが追加されました！」ポップアップが表示される

### そのほかの管理機能（admin.htmlにあるが今回の依頼範囲外）

- **見た目のカスタマイズ**: アクセントカラー・背景色・レアリティごとの色・演出ON/OFFを編集し、`theme.json`として書き出し→`data/theme.json`を上書き→push、で公開サイトのテーマを変更できる
- **データ管理**: カタログのJSONインポート、所持データ（コレクション進捗）のリセットが可能（いずれもこのブラウザ内のみに影響）
