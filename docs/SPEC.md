# FriendsGacha 仕様書（管理者用メモ）

このファイルは実装済みの仕様を自分用にまとめたものです。コードの詳細はソースを正としてください（各見出しに対応ファイルを記載）。

## 遊び部分

### ガチャの仕様

対応ファイル: [`js/gacha.js`](../js/gacha.js), [`js/rarity.js`](../js/rarity.js)

- **通貨**: マメ（[マメ工房](#マメ工房の仕様)で貯める）
- **消費コスト**: 1回引く = 5マメ / 10連引く = 50マメ
  - 10連は現在UI導線を非表示にしている（ロジック自体は残してあり、`#drawTenBtn`の`hidden`クラスを外せば復活可能）
- **抽選ロジック**:
  - レアリティを重み付き抽選（`RARITY`の`weight`: N=60 / R=30 / SR=8 / SSR=2、カタログに存在するレアリティのみで正規化）
  - 選ばれたレアリティ内のアイテムから完全ランダムで1点抽選
- **フロー**:
  1. 「引く」ボタン → カタログ読込 → マメ消費（不足していれば「マメが足りません」アラートで中断）→ コレクションに記録 → パックが開封待ち状態に
  2. パックをタップ → 開封演出 → カード表示 → 「もう一度引く」ボタン表示
  3. 「もう一度引く」を押すと、演出を挟まず**即座に次の1回を引き直す**（マメが足りなければアラートを出してパック選択画面に戻り、引くボタンは残高不足で無効化される）
- 抽選結果はブラウザのIndexedDB（`collection`ストア）に所持記録として保存される（`js/db.js`）

### マメ工房の仕様

対応ファイル: [`js/mame.js`](../js/mame.js)

- ブラウザごとの放置生産要素。IndexedDBの`settings`ストア（キー`'mame'`）に状態を保存
- **生産ライン**: 初期1本。5秒ごとに+1マメを自動生産（上限・バッファなし。何マメでも貯め続けられる）
- **タップ**: 生産ラインをタップすると、そのラインの残り時間が即座に短くなる
  - 初期状態は残り時間が「1/2」に短縮。「タップの効果」を強化するたびに 1/3 → 1/4 …と短縮率が強くなる
- **クリティカル**: 生産完了ごとに5%の確率で生産量が3倍になる（「✨ Critical!」トースト表示）
- **オフライン進行**: ページを開いた瞬間に前回訪問からの経過時間をまとめて計算し、その分のマメを即加算（オフライン中はクリティカル判定なし）
- **アップグレード**（マメを消費して強化、コストは購入するたびに指数的に上昇）:
  | アップグレード | 効果 | 初期コスト / 上昇率 |
  |---|---|---|
  | 製造速度 | 生産間隔を短縮（間隔 ×0.85 per Lv） | 15マメ / ×1.5 |
  | 一度に生産する量 | 1回の完成量アップ（1→2→5→10→20→50→100→200→500→1000、以降は倍々） | 25マメ / ×2 |
  | 並列生産ライン | ラインを1本追加（同時生産数が増える） | 150マメ / ×3 |
  | タップの効果 | タップ時の残り時間短縮率を強化 | 20マメ / ×1.8 |
- **今回のスコープ外**（意図的に未実装、将来の拡張候補）:
  - オーバーチャージ（100%超えまで待って回収すると倍率アップ）: 「完成したら即座に次の生産が始まる」という現行仕様と相性が悪いため見送り
  - バッファ上限（オフライン保管の上限）: 「上限なし」という方針のため未実装

### コレクションの仕様

対応ファイル: [`js/collection.js`](../js/collection.js)

- 達成率を「所持数 / 総アイテム数」で表示
- レアリティフィルタ（ALL / N / R / SR / SSR）
- 未所持アイテムはシルエット＋「？」表示、所持済みはサムネイル＋所持数バッジ表示
- アイテムをタップすると詳細モーダル（名前・作者・フレーバーテキスト・所持数・初取得日）を表示。未所持なら「？？？？？」と入手を促す文言のみ表示

### その他の共通UI

対応ファイル: [`index.html`](../index.html), [`js/main.js`](../js/main.js)

- ヘッダーの「？」ボタン → 遊び方ポップアップ（ガチャを引く／マメの製造／コレクションの3項目を説明）
  - **初回訪問時は自動的に開く**（IndexedDBの`settings`ストア、キー`'howToPlaySeen'`で既読管理）
- 新イラスト追加通知ポップアップ「🎉 新しいイラストが追加されました！」
  - `data/items.json`のアイテムID一覧をIndexedDB（キー`'knownItemIds'`）に記録し、前回訪問時より増えていたら表示
  - **遊び方ポップアップをまだ見ていない人（＝初回訪問者）には表示しない**（チュートリアルより新着通知が先に出るのを防ぐため）
- フッター: 左下にバージョン表記（`v1.0.0`固定テキスト、更新時は`index.html`を手動で書き換える）、右下にコピーライト

## 運営部分

### イラストの追加方法

対応ファイル: [`admin.html`](../admin.html), [`js/admin.js`](../js/admin.js)

1. `admin.html` をブラウザで開く（初回はブラウザのIndexedDBが空の場合、公開中の`data/items.json`を自動読み込みする）
2. フォームに入力して「追加する」を押す
   - 名前（必須）／レアリティ（必須、N・R・SR・SSR）／作者名／フレーバーテキスト／イラスト画像（必須、正方形に近い画像推奨）
   - 画像はBase64データURLとしてそのままアイテムデータに埋め込まれる（＝画像ファイルを別途サーバーに置く必要はないが、`items.json`が大きくなりがちなので画像サイズに注意）
   - **この時点ではまだこのブラウザ内（IndexedDB）にしか保存されておらず、公開サイトには反映されない**
3. 一覧から既存アイテムの「編集」「削除」も可能（同じくローカル反映のみ）
4. **公開サイトに反映する手順**:
   1. 「カタログを書き出す (items.json)」ボタンで`items.json`をダウンロード
   2. リポジトリの `data/items.json` をダウンロードしたファイルで上書き
   3. `git add` → `git commit` → `git push`（GitHub Pagesへの反映を確認）
5. 公開後、既にサイトを訪れたことがある（遊び方ポップアップ既読の）ユーザーには、次回アクセス時に自動で「新しいイラストが追加されました！」ポップアップが表示される

### そのほかの管理機能（admin.htmlにあるが今回の依頼範囲外）

- **見た目のカスタマイズ**: アクセントカラー・背景色・レアリティごとの色・演出ON/OFFを編集し、`theme.json`として書き出し→`data/theme.json`を上書き→push、で公開サイトのテーマを変更できる
- **データ管理**: カタログのJSONインポート、所持データ（コレクション進捗）のリセットが可能（いずれもこのブラウザ内のみに影響）
